مرحله ۲ — شناسه‌ها و آدرس Worker

با هر بار تولید Token جدید، کد Worker بالا نیز به‌صورت خودکار آپدیت می‌شود.

این مقدار برای ساخت کانفیگ‌های پروتکل Trojan استفاده می‌شود — همانند Token، با تغییر آن در کد Worker، کانفیگ‌های Trojan قدیمی از کار می‌افتند. با هر بار تولید Password جدید، کد Worker بالا نیز به‌صورت خودکار آپدیت می‌شود.

⚠
فقط نام دامنه را وارد کنید — مثال: myworker.username.workers.dev — در صورت وارد کردن https:// یا / اضافه، به‌صورت خودکار اصلاح می‌شود.

مرحله ۳ — IP های سالم و تنظیمات پیشرفته
ℹ
برای یافتن IP های سالم Cloudflare می‌توانید از ابزار Clean IP Scanner استفاده کنید. این ابزار مبتنی بر Termux بوده و روی اندروید اجرا می‌شود — نتیجه اسکن را مستقیم در کادر زیر وارد کنید.

هر تعداد IP قابل قبول است — از هر IP چند کانفیگ با پورت‌های مختلف ساخته می‌شود. علاوه بر IP، می‌توانید از برخی دامنه‌ها نیز استفاده کنید (مثلاً chatgpt.com)

می‌توانید فقط VLESS، فقط Trojan یا هر دو را همزمان فعال کنید — تمام کانفیگ‌های Link و فایل‌های JSON/YAML بر اساس همین انتخاب ساخته می‌شوند.

در صورت فعال بودن، تنظیمات fragment فقط در کانفیگ مادر JSON اعمال می‌شود و روی کانفیگ‌های VLESS تأثیری ندارد.

فعال کردن چک‌باکس کافی است — Xray کلید رمزنگاری دامنه Worker شما را به صورت خودکار از این سرور DNS دریافت می‌کند. ECH روی کانفیگ‌های JSON و URL (فرمت vless://) اعمال می‌شود. آدرس پیش‌فرض (DoH Cloudflare) بهترین گزینه برای ایران است زیرا یک آدرس https:// رمزگذاری‌شده است و برخلاف udp://8.8.8.8 در برابر دستکاری DNS توسط ISPها مقاوم است. اگر DoH Proxy شخصی دارید، آدرس آن را اینجا وارد کنید.

Local DNS برای resolve دامنه‌های ایرانی استفاده می‌شود. Remote DNS به‌صورت پیشفرض روی Cloudflare DoH تنظیم است و می‌توانید آن را با هر آدرس DoH دلخواه جایگزین کنید. با ابزار DoH Proxy می‌توانید یک سرور DoH اختصاصی روی Cloudflare بسازید و آدرس آن را اینجا وارد کنید. Allow LAN اتصال سایر دستگاه‌های شبکه محلی را فعال می‌کند. ECH فقط در حالت Normal (بدون Fragment) روی کانفیگ JSON و کانفیگ‌های فرمت URL اعمال می‌شود.

ℹ
هر ترکیب پورت یک کانفیگ مجزاست. با انتخاب بیشتر، کانفیگ‌های متنوع‌تری برای دور زدن فیلترینگ هوشمند ساخته می‌شه. پیشنهاد: ابتدا فقط پورت 443 را تست کنید و اگر کار نکرد پورت‌های دیگر را هم فعال کنید.

اگر خالی بماند، پنل نام پیش‌فرض را برای کانفیگ Normal یا Fragment انتخاب خواهد کرد.

کانفیگ JSON — Sing-box 📦
TCB_Singbox.json

این کانفیگ مستقیماً برای Sing-box و کلاینت‌های مبتنی بر هسته‌ی Sing-box ساخته شده. به دلیل Deprecated شدن فیلد ECH در نسخه فعلی Sing-box، این کانفیگ بدون ECH ساخته می‌شود (Fragment و سایر تنظیمات شما اعمال شده‌اند).

کانفیگ JSON — Clash 🐱
TCB.yaml

فرمت و دانلود این کانفیگ با پسوند .yaml انجام می‌شود تا کلاینت‌های Clash/Mihomo آن را به‌عنوان پروفایل بشناسند. توجه: فرمت Clash/Mihomo اصولاً قابلیت Fragment ندارد، بنابراین تنظیمات Fragment روی این کانفیگ اعمال نمی‌شود.
